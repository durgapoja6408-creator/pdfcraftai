## Findings

### SEV-2 — `/account` is dead and caches a headerless 307 at the CDN

**Evidence:**
```
$ curl -I https://pdfcraftai.com/account
HTTP/2 307
x-nextjs-cache: HIT
cache-control: s-maxage=31536000, stale-while-revalidate
  (no Location header)
```

- `/pricing` links to `/account` ("Configure BYOK" button), but the `/account`
  route was never shipped. Next's `not-found` boundary serves a 307 that gets
  frozen by the full-year `s-maxage` at Cloudflare, stripping the `Location`
  header along the way.
- Same symptom pattern as the old `/signup` bug (task #71); same fix.
- Visible to authenticated users too (confirmed with a real session cookie),
  so it's not auth-dependent — it's cached CDN state.

**Fix applied** (this commit): add `app/account/page.tsx` that server-redirects
to `/app/settings` with `dynamic = "force-dynamic"` to skip the static prerender cache.
Matches the `/signup` → `/register` pattern.

### SEV-3 — 9 marketing pages missing `og:url`

Pages without `og:url` in `<head>`: `/about`, `/contact`, `/gdpr`,
`/careers`, `/status`, `/changelog`, `/bulk`, `/macros`, `/agent`.

Cause: each page sets `alternates.canonical` but omits an `openGraph.url`
override, so Next.js's default merge doesn't emit the tag. Compare with
`/tools` which does set `openGraph.url` and ships correctly.

**Fix applied** (this commit): add `openGraph: { url: "/<slug>" }` to each of
the 9 pages' `metadata` exports.

### Known open items (unchanged)

- **Task #72 [blocked on user]**: `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` still
  missing from Hostinger env. AI routes gate correctly at 401 for anon, but
  authed POSTs will 500 until a key is set.
- **Task #13 [blocked on #72]**: AI tool end-to-end tests.
- **Task #14 [pending]**: P3 settings, billing UI, destructive flows.
